AI MediaKit 剧本还原工具基于强大的视频理解能力，能够将真人实拍的短剧转化为结构化剧本文本。本工具可以精准识别并提取视频中的场景、人物（角色）、对话、情节等核心元素，为您提供一份格式清晰、内容完整的剧本文件，极大地提升了内容分析、二次创作和检索的效率。

本篇开发指南将引导您：


* 快速体验通过 API 调用，将剧集视频还原为结构化的剧本文本。

* 理解并使用剧本还原的异步任务流程。

* 了解剧本还原工具的适用场景与输入要求。

## 快速入门：实现剧本还原

<span id="tlQk3DPvgX"></span>
### 准备工作


1. **获取访问凭证**：在开始之前，访问 AI MediaKit 控制台 [API Key 管理页面](https://console.volcengine.com/imp/ai-mediakit/settings)，创建并获取 **AI MediaKit API Key**。

2. **准备视频文件**：准备一个或多个待分析的剧集视频文件，并将其上传至公网可访问的服务器或[视频点播服务](https://www.volcengine.com/docs/4/2834?lang=zh)中，以获取可供 API 调用的 URL。


<span id="d4lBei6C9z"></span>
### 操作流程

剧本还原是一个异步任务，完整的调用流程如下：


1. 成功调用 `POST /api/v1/tools/drama-script` 提交任务后，API 将返回一个任务 ID。

2. 您可以轮询 `GET /api/v1/tasks/{task_id}` 接口查询任务状态，直到状态 `status` 变为 `completed`；或者使用[事件回调](https://www.volcengine.com/docs/6448/2288701)自动接收任务状态变化。

3. 任务完成后，从响应的 `result` 字段中获取剧本文件下载地址 (`result_url`)，并下载最终的 JSON 文件。


<span id="rPwlT79QzE"></span>
#### 第 1 步：提交剧本还原任务

您需要调用[提交剧本还原任务 API](https://www.volcengine.com/docs/6448/2484442)。可直接复制以下 cURL 命令，提交一个剧本还原任务。本示例输入了两个剧集视频文件。

```Bash
curl -X POST 'https://mediakit.cn-beijing.volces.com/api/v1/tools/drama-script' \
-H 'Authorization: Bearer {Your_API_Key}' \
-H 'Content-Type: application/json' \
-d '{
  "video_urls": [
    "https://example.com/drama_episode_01.mp4",
    "https://example.com/drama_episode_02.mp4"
  ]
}'
```


<div data-tips="true" data-tips-type="default" data-tips-is-title="true">说明</div>


<div data-tips="true" data-tips-type="default"><strong>需替换参数</strong>：</div>



* <div data-tips="true" data-tips-type="default">将 <code>{Your_API_Key}</code> 替换为您的有效 API Key。</div>


* <div data-tips="true" data-tips-type="default">将 <code>video_urls</code> 列表中的 URL 替换为您自己的视频文件地址。</div>



请求成功后，您会收到如下包含 `task_id` 的响应，请保存此 ID 用于后续查询。

```JSON
{
  "success": true,
  "task_id": "amk-tool-drama-script-123456789",
  "request_id": "20240522150000ABCDEFG12345"
}
```


<span id="EEeEJJTFw9"></span>
#### 第 2 步：获取任务结果

剧本还原需要一定处理时间，您需要通过轮询[查询任务信息 API](https://www.volcengine.com/docs/6448/2278532) 来获取最终结果，直到任务状态变为 `completed`。

```Bash
curl -X GET 'https://mediakit.cn-beijing.volces.com/api/v1/tasks/{task_id}' \
-H 'Authorization: Bearer {Your_API_Key}'
```


<div data-tips="true" data-tips-type="default" data-tips-is-title="true">说明</div>


<div data-tips="true" data-tips-type="default"><strong>需替换参数</strong>：将 URL 末尾的 <code>{task_id}</code> 替换为第 1 步返回的任务 ID。</div>


任务成功完成后，您将收到包含剧本文件下载地址的响应。`result.result_url` 即是最终生成的剧本文件下载链接。该 URL 有效期为 **24 小时**，请务必及时保存。

```JSON
{
    "success": true,
    "task_id": "amk-tool-drama-script-123456789",
    "status": "completed",
    "result": {
        "result_url": "https://example.volcvideo.com/script.json.gz?auth_key=...",
        "duration": 245.5
    },
    "created_at": 1777291767,
    "finished_at": 1777291851,
    "expires_at": 1777464650
}
```


<span id="P4IOAoq1la"></span>
#### 第 3 步：下载并使用剧本数据

访问第 2 步中获取的 `result_url`，即可下载一个 Gzip 压缩的 JSON 文件。根据您在提交任务时 `return_pkg` 参数的设置，产物格式会有所不同。

**默认情况 (return_pkg=false)：获取核心剧本数据**

默认情况下，您将下载一个 Gzip 压缩的 JSON 文件 (result.json.gz)。解压后，即可得到一份包含完整结构化剧本数据的 JSON 文件。

**完整包情况 (return_pkg=true)：获取全部解析资产**

如果您在提交任务时设置了 `return_pkg: true`，下载的将是一个包含所有分析产物的压缩包 (.tar.gz)。这对于深度二次开发非常有用。解压后的文件结构如下：

```Plain Text
.tar.gz
├── result.json         # 核心产物：结构化的剧本还原结果
├── faces/              # 人物聚类图片，按识别到的人物命名文件夹
│   ├── 人物A/
│   │   ├── 12.jpg
│   │   └── 35.jpg
│   └── 人物B/
│       └── 48.jpg
├── scene_frames/       # 按场景划分的视频帧截图
    ├── 000/
    │   ├── 0001.jpg
    │   └── 0002.jpg
    └── ...
```

## 参考信息

<span id="300cb517"></span>
### 使用限制

为确保最佳还原效果，请注意以下输入限制：


|限制项 |说明 |
|---|---|
|**内容类型** |适用于**真人实拍的短剧、长剧、电影**等以人物对话和情节发展为核心的视频内容。不适用于**动画、纪录片、广告、直播录屏**等缺乏连贯真人剧情或人脸识别线索的视频。 |
|**字幕要求** |视频必须包含清晰的**内嵌字幕（硬字幕）** 。 |
|**分辨率** |提交的多个视频，其分辨率（宽高）必须保持一致。 |
|**数量与时长** |*单次任务支持传入 **1 到 100 个**视频文件。<br><br>* 单个文件时长不超过 **120 分钟**。<br><br>* 所有视频累计总时长不超过 **300 分钟**。 |
|**视频格式** |支持 `mp4`, `mov`, `mkv` 等主流视频格式，暂不支持 HLS (M3U8) 格式。 |
