`POST https://mediakit.cn-beijing.volces.com/api/v1/tools/video-understand-router`

本 API 用于提交一个异步的视频理解智能策略任务，基于视觉大模型，对输入的视频 URL 列表进行通用视频内容分析，输出视频级别的结构化理解结果，适用于内容审核、视频检索、标签生成等场景。


* **获取任务结果**：任务提交成功后，您需要保存返回的 `task_id`，并通过轮询[查询任务信息 API](https://www.volcengine.com/docs/6448/2278532) 或配置[事件回调](https://www.volcengine.com/docs/6448/2288701)获取最终结果。

* **开发指南**：如需了解完整的使用流程和应用示例，请参阅[视频理解智能策略](https://www.volcengine.com/docs/6448/2551133)。

## 请求参数

<span id="2eab8223"></span>
### Header 参数


|参数 |类型 |是否必选 |示例值 |描述 |
|---|---|---|---|---|
|`Authorization` |String |是 |`Bearer {Your_API_Key}` |格式为 `Bearer {Your_API_Key}`。请参考[基础概念及准备工作](https://www.volcengine.com/docs/6448/2300661)获取 API Key。 |


<span id="aa6ff5fb"></span>
### Body 参数


|参数 |类型 |是否必选 |示例值 |描述 |
|---|---|---|---|---|
|`video_urls` |Array of String |是 |`["https://example.com/video1.mp4", "https://example.com/video2.mp4"]` |待处理的视频 URL 列表。<br><br><br>*视频来源：支持公网 HTTP/HTTPS URL、本地上传 (mediakit://)、来源于火山引擎视频点播 (vod://) 和火山引擎对象存储 (tos://) 四种输入协议。关于各协议的具体说明和使用示例，请参阅[多源媒体输入与本地上传](https://www.volcengine.com/docs/6448/2536893)。<br><br>* 支持格式：mp4, flv, ts, avi, mov, wmv, mkv 等主流视频格式。<br><br>*数量限制：单次任务最多支持传入 10 个视频文件。<br><br>* 单视频时长限制：单个视频时长不超过 2 小时。 |
|`prompt` |String |是 |`请总结视频中的关键事件，并标注每个事件出现的时间段` |提示词，用于指导大模型对视频内容进行分析的自然语言描述。最小长度为 1。 |
|`level` |String |否 |`Balanced` |分析档位。该参数会决定任务的默认抽帧策略与模型选择，以在成本、速度和质量之间取得平衡。详细的决策逻辑，请参阅[工作原理](https://www.volcengine.com/docs/6448/2551133#7d257d62)。取值如下：<br><br><br>*`Economy`：（默认）经济档位，速度优先，适合大批量、对结果详细程度要求较低的内容标注。<br><br>* `Balanced`：均衡档位，速度与质量兼顾，适合常规内容审核与检索场景。<br><br>* `Quality`：质量档位，结果优先，适合需要更精细语义理解的场景。 |
|`prefer_models` |Array of String |否 |`["doubao-seed-2-0-lite-260428"]` |优先使用的**模型 ID (Model ID)**  列表，最多 10 个。系统将从您指定的模型中结合策略选择最终模型。您可以在[火山方舟控制台在线推理页面](https://console.volcengine.com/ark/region:ark+cn-beijing/endpoint)查看可用的模型 ID。 |
|`prefer_endpoints` |Array of String |否 |`["ep-m-20260703202744-q8tcb"]` |优先使用的**推理接入点 ID (Endpoint ID)**  列表，最多 10 个。系统将从您指定的推理接入点中结合策略选择最终模型。您可以在[火山方舟控制台在线推理页面](https://console.volcengine.com/ark/region:ark+cn-beijing/endpoint)创建和管理推理接入点。<br><br>**注意**：此参数优先级高于 `prefer_models`。 |
|`manual_option` |Object |否 |`-` |手动模式相关参数。详见下文 [manual_option 结构](https://www.volcengine.com/docs/6448/2552748#08054294)。该字段未传时表示完全使用 `level` 档位策略，不进行手动覆盖。 |
|`client_token` |String |否 |`your_unique_request_id_131` |用户请求凭证，用于幂等控制。大小写敏感，不超过 64 个 ASCII 码可打印字符。详情请参考[幂等性控制](https://www.volcengine.com/docs/6448/2300661#672adae7)。 |
|`callback_args` |String |否 |`your_unique_callback_args` |自定义回调参数。您提供的内容将在任务完成时，通过[事件回调](https://www.volcengine.com/docs/6448/2288701)原样返回，方便您关联业务。字段长度最大为 512 字节。 |
|`callback_url` |String |否 |`https://example.com/callback` |用于接收该任务结果回调的 URL 地址。如果提供此参数，其优先级将高于在控制台配置的全局回调地址。地址必须以 http:// 或 https:// 开头。请参考[事件回调](https://www.volcengine.com/docs/6448/2288701)了解回调接收机制和回调消息结构。 |
|`queue_id` |String |否 |`q00a9731598092652e4a95292434198` |任务提交的目标队列 ID。如不传，默认会使用系统自动创建的队列 ID。您可以将不同业务或优先级的任务提交到不同的队列，以实现按队列对应的项目进行分账。您可以在 AI MediaKit 控制台队列管理页面创建和管理队列，系统会自动分配一个队列 ID。详情请参考[项目与队列管理](https://www.volcengine.com/docs/6448/2519520)。 |


<span id="08054294"></span>
#### manual_option 结构


|参数 |类型 |是否必选 |示例值 |描述 |
|---|---|---|---|---|
|`max_snapshot_number` |Integer |否 |`100` |最大截图帧数，取值范围 [0, 1000]，默认 0。设为 0 时由 `level` 档位决定截图数量；显式设置时会覆盖档位策略。 |
|`need_audio` |Boolean |否 |`false` |是否需要识别音频。默认 `false`。设为 `true` 时会同时分析音频内容（语音、台词、声音特征等）。<br><br>是否强制开启或关闭音频分析，默认为 `false`。<br><br><br>*`true`：强制开启音频分析，系统将选用支持音视频多模态的模型。<br><br>* `false`：强制关闭音频分析，仅分析视频画面。<br><br><br>**注意**：即使此参数为 `false` 或未提供，如果 `prompt` 中包含“声音”、“音乐”等音频相关关键词，系统可能也会自动触发音频分析。 |

## 请求示例

以下示例演示了如何对一个公网视频进行内容分析，要求对视频内容做总结。

```Bash
curl -X POST 'https://mediakit.cn-beijing.volces.com/api/v1/tools/video-understand-router' \
  -H 'Authorization: Bearer {Your_API_Key}' \
  -H 'Content-Type: application/json' \
  -d '{
    "video_urls": [
      "https://example.com/sample_video.mp4"
    ],
    "prompt": "请总结视频中的关键事件，并标注每个事件出现的时间段",
    "level": "Balanced",
    "manual_option": {
      "max_snapshot_number": 100,
      "need_audio": true
    }
  }'
```

## 响应说明

<span id="a7016aa7"></span>
### 任务提交成功响应


|参数 |类型 |描述 |
|---|---|---|
|`success` |Boolean |任务是否提交成功。<br><br><br>*`true`：成功。<br><br>* `false`：失败，更多信息请查看[错误处理](https://www.volcengine.com/docs/6448/2552748#410613a4)。 |
|`task_id` |String |任务的唯一标识，用于后续查询任务状态和结果。 |
|`request_id` |String |本次请求的唯一标识，可用于问题排查。 |


示例：

```JSON
{
  "success": true,
  "task_id": "amk-tool-video-understand-router-123456789",
  "request_id": "20260624150000ABCDEFG12345"
}
```


<span id="a932d65b"></span>
### 异步任务结果

任务处理完成后，通过[查询任务信息 API](https://www.volcengine.com/docs/6448/2278532) 或[事件回调](https://www.volcengine.com/docs/6448/2288701)获取的 `result` 字段将包含以下内容。


|参数 |类型 |描述 |
|---|---|---|
|`duration` |Number |输入视频总时长，单位为秒，为所有输入视频时长的总和。 |
|`contents` |Array of String |视频内容分析结果列表。每一项对应 `video_urls` 中同索引视频的分析结果文本。 |
|`token_usage` |Object |大模型推理过程中的 Token 使用量统计。详见下文 [token_usage 结构](https://www.volcengine.com/docs/6448/2552748#7fc3c2a4)。 |


<span id="7fc3c2a4"></span>
#### token_usage 结构


|参数 |类型 |描述 |
|---|---|---|
|`input_tokens` |Integer |输入 Token 数量。 |
|`output_tokens` |Integer |输出 Token 数量。 |
|`total_tokens` |Integer |总 Token 数量。 |


最终的完整任务信息示例：

    ],
    "token_usage": {
      "input_tokens": 850,

      "total_tokens": 1000
    }
  },
  "expires_at": 1782001650,
  "created_at": 1781828767,

## 错误处理

当请求的参数或鉴权信息不正确时，任务将不会被创建，接口会返回一个同步的错误响应。详见[错误码](https://www.volcengine.com/docs/6448/2300662)。示例如下：

```JSON
{
  "success": false,
  "task_id": "",
  "request_id": "20260624150500XYZW12345",
  "error": {
    "code": "InvalidParameter",
    "message": "The parameter `video_urls` exceeds the maximum length of 10.",
    "param": "video_urls",
    "type": "BadRequest"
  }
}
```
