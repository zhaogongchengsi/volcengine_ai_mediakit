AI MediaKit 的解说视频生成（短剧行业模型）工具，基于对输入短剧剧集的角色和剧情故事线理解（剧本还原），自动提取高光片段并按剪辑要求（时长范围、产出条数、解说浓度、BGM 等）生成全新的解说视频（支持**文字解说**和**旁白解说**两种模式），适用于短剧投流、宣发种草、二创剪辑等场景。

* 理解短剧行业模型的工作流程与核心优势。

* 掌握如何通过各项参数，精细化地控制解说模式、时长、数量、视听风格等。

## 快速体验：生成一个文字解说视频

本节将演示如何通过一次 API 调用，从多个原始短剧视频快速生成一个文字解说视频。

<span id="HY6Z1RnisB"></span>
### 准备工作


1. **获取访问凭证**：在开始之前，访问 AI MediaKit 控制台 [API Key 管理页面](https://console.volcengine.com/imp/ai-mediakit/settings)，创建并获取 **AI MediaKit API Key**。

2. **准备视频文件**：准备一个或多个短剧视频，并将其上传至公网可访问的服务器或[视频点播服务](https://www.volcengine.com/docs/4/2834?lang=zh)中，以获取可供 API 调用的 URL。为方便您快速测试，本文后续步骤的示例代码中提供了两个可直接使用的演示视频 URL。


<span id="U63rheUxwv"></span>
### 操作流程

与通用版的解说视频生成不同，短剧行业模型将“剧本分析”和“视频生成”合二为一，流程更简单。核心步骤如下：


1. **提交解说视频生成任务**：调用 `POST /api/v1/tools/drama-recap-vertical` API，传入原始视频地址和相关配置，获得一个用于查询的 `task_id`。

2. **获取最终结果**：轮询 `GET /api/v1/tasks/{task_id}` 直至任务完成。任务成功后，从响应的 `result` 字段中获取最终生成的解说视频下载地址。


<span id="a7KmaBChJt"></span>
### 操作步骤

<span id="LpQGyqU5uz"></span>
#### 第 1 步：提交解说视频生成任务

您可以直接复制以下 cURL 命令，提交一个异步的解说视频生成任务。该示例将生成一个不带背景音乐的旁白解说风格视频。

```Bash
curl -X POST 'https://mediakit.cn-beijing.volces.com/api/v1/tools/drama-recap-vertical' \
-H 'Authorization: Bearer {Your_API_Key}' \
-H 'Content-Type: application/json' \
-d '{
  "video_urls": [
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_1.mp4",
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_2.mp4",
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_3.mp4"
  ],
  "mode": "text",
  "max_count": 1,
  "max_duration": 60,
  "opening_hook": "force"
}'
```


<div data-tips="true" data-tips-type="default" data-tips-is-title="true">说明</div>



* <div data-tips="true" data-tips-type="default">将 <code>{Your_API_Key}</code> 替换为您的真实 API Key。</div>


* <div data-tips="true" data-tips-type="default"><code>video_urls</code>: 示例中已提供可直接测试的视频 URL。您可以替换为您自己的地址。</div>


* <div data-tips="true" data-tips-type="default"><code>mode: "text"</code> 代表生成文字解说视频。该模式下，AI 生成的解说文案将以文字形式呈现在画面上。</div>



请求成功后，您会收到一个用于查询进度的 `task_id`（例如 `amk-tool-drama-recap-vertical-12345`）。

<span id="Ulx09uGfZV"></span>
#### 第 2 步：轮询获取任务结果

接下来，您需要使用上一步获得的 `task_id` 轮询[查询任务信息 API](https://www.volcengine.com/docs/6448/2278532)，直到任务状态 `status` 变为 `completed`。

```Bash
curl -X GET 'https://mediakit.cn-beijing.volces.com/api/v1/tasks/amk-tool-drama-recap-vertical-12345' \
-H 'Authorization: Bearer {Your_API_Key}'
```


当任务成功完成后，您将收到如下响应。您可以在 `result.video_urls` 数组中找到最终生成的解说视频下载地址。

```JSON
{
    "success": true,
    "task_id": "amk-tool-drama-recap-vertical-12345",
    "request_id": "20240522180000ABCDEFG12345",
    "result": {
        "input_duration": 245.5,
        "output_duration": 58.2,
        "mode": "text",
        "video_urls": [
            "https://example.volcvideo.com/narration_output_1.mp4?auth_key=..."
        ],
        "video_infos": [
            {
                "duration": 58.2,
                "size": 48123456,
                "poster_url": "",
                "video_url": "https://example.volcvideo.com/narration_output_1.mp4?auth_key=..."
            }
        ]
    },
    ...
}
```

## 进阶使用

在快速入门的基础上，您可以通过配置不同的参数，实现更丰富的创作效果。

<span id="scm8gwyP7w"></span>
### 自定义旁白解说效果

当 `mode` 设置为 `narrate` 时，您可以通过 `narrate_options` 和 `narrate_bgm_url` 控制旁白解说的效果。


* `narrate_options.narrate_ratio`：解说浓度。控制旁白时长占生成视频时长的比例，取值范围为 `[0, 1]`。值越大，解说越密集。建议设置为 `0.3` 到 `0.5` 之间。

* `narrate_options.enable_narrate_bgm`：是否添加背景音乐，默认为 `true`。

* `narrate_bgm_url`：当 `narrate_options.enable_narrate_bgm` 为 `true` 时，自定义背景音乐的 URL。


<span id="KUXYYiOIEb"></span>
#### 示例：设置较高解说浓度并添加自定义 BGM

以下示例将生成一个旁白解说密集且添加背景音乐的视频。

```Bash
curl -X POST 'https://mediakit.cn-beijing.volces.com/api/v1/tools/drama-recap-vertical' \
-H 'Authorization: Bearer {Your_API_Key}' \
-H 'Content-Type: application/json' \
-d '{
  "video_urls": [
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_1.mp4",
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_2.mp4",
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_3.mp4"
  ],
  "mode": "narrate",
  "narrate_bgm_url": "https://vod-ai-test.tos-cn-beijing.volces.com/demo/bgm.mp3",
  "max_count": 1,
  "max_duration": 60,
  "narrate_options": {
    "narrate_ratio": 0.5,
    "enable_narrate_bgm": true
  }
}'
```


<span id="PZQdSmfruK"></span>
### 自定义文字解说样式

当 `mode` 设置为 `text` 时，您可以通过 `text_options` 对象精细化控制屏幕解说文字的样式。


* `font_type`：字体，如 `SY_Bold` (思源粗体)。

* `font_size`：字号。

* `font_color`：字体颜色，格式为 `#RRGGBBAA`。

* `border_width` / `border_color`：描边宽度和颜色。

* `shadow_color`：阴影颜色。

* `is_bold` / `is_italic`：是否加粗/斜体。


<span id="eon62S145B"></span>
#### 示例：生成一个自定义样式的文字解说视频

以下示例将生成一个“文字解说”风格的视频，并自定义了解说文字的样式（如黄色、带描边等）。

```Bash

-H 'Authorization: Bearer {Your_API_Key}' \
-H 'Content-Type: application/json' \
-d '{
  "video_urls": [
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_1.mp4",
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_2.mp4",
    "https://vod-ai-test.tos-cn-beijing.volces.com/demo/drama_3.mp4"
  ],
  "mode": "text",
  "max_count": 1,
  "max_duration": 60,
  "text_options": {
    "font_type": "SY_Bold",
    "font_size": 45,
    "font_color": "#FFFF00FF",
    "border_width": 2,
    "border_color": "#000000FF",
    "is_italic": false
  }
}'
```

### 控制产出数量与时长

您可以控制单次任务生成的视频数量和每个视频的长度范围。


* `max_count`：期望生成的视频**数量上限**。默认 `3`，范围 `[1, 100]`。

* `min_duration`：单个视频的**最小时长**（秒）。默认 `30`。

* `max_duration`：单个视频的**最大时长**（秒）。默认 `180`。


<span id="TEZPLbENes"></span>
### 开启精彩片段前置

为了在视频开头就抓住用户注意力，您可以通过 `opening_hook` 参数控制是否将全剧最精彩的片段剪辑到视频开头。


* `auto`：（默认）智能判断是否开启。

* `force`：强制开启。

* `disable`：关闭。

## 使用限制


|限制项 |说明 |
|---|---|
|输入视频限制 |***视频来源**：支持公网 HTTP/HTTPS URL、本地上传 (`mediakit://`)、来源于火山引擎视频点播 (`vod://`) 和火山引擎对象存储 (`tos://`) 四种输入协议。关于各协议的具体说明和使用示例，请参阅[多源媒体输入与本地上传](https://www.volcengine.com/docs/6448/2536893)。<br><br>* **支持格式**：`mp4`, `flv`, `ts`, `avi`, `mov`, `wmv`, `mkv` 等主流视频格式。<br><br>***数量限制**：单次任务支持传入 1 到 30 个视频文件。<br><br>* **时长限制**：单次任务处理的所有视频累计时长不得超过 120 分钟（2 小时）。<br><br>***输入分辨率限制**：当前仅支持 1080p。输入素材需保持分辨率一致，否则会有兼容性问题。<br><br>* **输入内容要求**：视频必须同时包含视频流和音频流<br><br>   ***音频流要求**：音频轨道中必须包含清晰可识别的中文对话文本，仅含 BGM/纯音乐/语气词的片段无法准确还原剧情。<br><br>* **画面字幕**：建议视频画面下半部分包含清晰居中的中文字幕，可显著提升文字解说定位与剧情理解准确度。 |
|输出视频数量 |支持最多生成 100 个解说视频。 |
|输出视频时长 |输出视频的时长限制在 1 到 7200 秒之间。 |
|任务处理时长 (RTF) |任务处理时长与输入视频总时长相关。RTF（Real\-Time Factor）指“任务处理耗时 / 输入视频总时长”。<br><br><br>***文字解说模式**：RTF 通常在 3.5 到 5.0 之间。<br><br>* **旁白解说模式**：RTF 通常在 2.5 到 3.5 之间。<br><br><br>此数值仅供参考。当任务并发量较高时，任务处理可能会进入排队状态，实际耗时会相应增加。 |
